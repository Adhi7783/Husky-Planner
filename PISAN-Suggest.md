## Project Summary

Husky Planner is a React 18 + TypeScript + Vite single-page app that lets a UW student track classes and assignments and ask an LLM (Groq, model `llama-3.1-8b-instant`) to rank incomplete assignments by priority. State is held in a Zustand store and persisted to `localStorage`. - The prompt at [`geminiService.ts:57-82`](src/services/geminiService.ts#L57-L82) only sees `name`, `dueDate`, `className`, and optional `description`. ### New Features

- **Canvas / MyPlan import.** The biggest UW-community wins all come from removing manual entry. Pull a student's current quarter assignments via Canvas API and their schedule via MyPlan or a UW time schedule scrape. This single change would convert the app from "generic to-do" to "actually UW-specific."
- **All-Upcoming cross-class view** on the dashboard, grouped Overdue / Today / This Week / Later. The single most common planner workflow and works without an API call.
- **Per-class color tag.** Add `color` to `Class` in [`types/index.ts:1-5`](src/types/index.ts#L1-L5) with a swatch picker; class chips, priority items, and the upcoming list all inherit. Cheap and high readability.
- **Recurring assignments** (weekly problem sets, biweekly labs). Dominant pattern in UW courses.
- **Edit class / edit assignment** flow. Today only add / toggle-complete / delete exist; correcting a typo forces a delete + re-add that loses the AI explanation.
- **`.ics` export.** Emit assignments as an iCal feed so students can subscribe in Google / Apple Calendar. Pairs naturally with per-user storage.
- **Quick-add via natural language**, e.g. "PS3 for CSE 142 due Friday 11:59pm" parsed into class + name + due date. Falls back to the structured form.

### AI Integration Enhancements

- **Feed the AI richer signals.** Add an `estimatedMinutes` field on `Assignment`, an `instructorWeight` or assignment type ("project" | "ps" | "quiz" | "reading"), and recent completion behaviour. The current prompt at [`geminiService.ts:57-82`](src/services/geminiService.ts#L57-L82) asks the model to reason about "workload" and "dependencies" but receives neither.
- **Weekly schedule planner.** Beyond static ranking, ask the model to allocate assignments to time blocks across the next 7 days given an `estimatedMinutes` field and a student-declared availability window per day. This is the kind of "AI is central" feature the rubric is asking for.
- **Syllabus ingestion.** Let a student paste or upload a syllabus PDF; an LLM extracts assignment list, due dates, weights, and seeds the planner. Solves the cold-start data-entry problem and is a defensible "natural language processing for data extraction" use case from the spec.
- **Explain-then-edit interaction.** When the AI explains "Due soon and likely high workload," let the student click "actually low workload" and have that feed back into the next prompt as a one-shot example. Creates a tight, AI-aware feedback loop.
- **Streaming responses.** Use Groq's `stream: true` so explanations appear progressively; the current 30s blocking call ([`geminiService.ts:34`](src/services/geminiService.ts#L34)) feels heavy for a list of ~10 items.
